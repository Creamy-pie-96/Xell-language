# Autocomplete & Code Suggestion Plan

## Principle: DRY — Single Source of Truth

All completion data flows from C++ source → `gen_xell_grammar.py` → JSON files → xell-terminal loads at runtime.
**Zero hardcoded keyword/builtin lists in the IDE.** Only the JSON parser and rendering live in C++.

### Data pipeline

```
src/lexer/token.hpp           ─┐
src/builtins/builtins_*.hpp    ├─→ gen_xell_grammar.py ─┬─→ language_data.json  (keywords, builtins, hover docs)
src/builtins/register_all.hpp ─┘                        ├─→ xell.json           (snippets with tab stops)
                                                        └─→ terminal_colors.json (already used by highlighter)
```

The IDE loads `language_data.json` + `xell.json` at startup and uses them for all completions.
When builtins or keywords change, run `gen_xell_grammar.py` — everything updates.

### Phase 2: Snippet Engine — `src/ui/snippet_engine.hpp` (NEW)

Parse `xell.json` (VSCode snippet format) at runtime.

**Data structures:**

```
SnippetDef {
    prefix: string          # trigger text ("fn", "for", "ife")
    body: vector<string>    # lines with ${N:placeholder} tab stops
    description: string
}

ActiveSnippet {
    lines: vector<string>               # expanded text
    tabStops: vector<TabStop>            # {line, col, length, placeholder}
    currentStop: int                     # which tab stop is active
    startLine, startCol: int             # where snippet was inserted
}
```

**Tab stop parsing:** `${1:name}` → stop index 1, placeholder "name", `$0` = final cursor.

**Operations:**

- `loadSnippets(path)` — parse xell.json
- `expand(prefix, insertLine, insertCol)` → ActiveSnippet
- `nextTabStop()` / `prevTabStop()` → move between fillable parts
- `commitSnippet()` — finalize, remove tab stop tracking

### Phase 3: Ghost Text — `src/editor/editor_view.hpp` changes

Dim/grey text rendered after the cursor showing the top completion.

**In renderInto():**

- After rendering the current line, if ghost text is set:
  - Render remaining ghost chars in dim color (`{100, 100, 100}`) after cursor
  - Ghost text = `topCompletion.label.substr(prefix.length())`

**Data:**

```
ghostText_: string        # the dim suffix to show
ghostLine_: int           # which buffer line it applies to
ghostCol_: int            # column where ghost starts
```

**When ghost text is active:**

- Tab → accept ghost text (insert into buffer)
- Any other key → ghost disappears, autocomplete re-evaluates
- Escape → dismiss

### Phase 4: Integration — `src/ui/layout_manager.hpp` wiring

**New members:**

```cpp
CompletionDB completionDB_;
AutocompletePopup autocompletePopup_;
SnippetEngine snippetEngine_;
std::string acPrefix_;           // current completion prefix
uint32_t acLastKeyTime_ = 0;    // for debounce
bool acManualTrigger_ = false;   // Ctrl+Space
```

**Initialization (in constructor / init):**

```cpp
completionDB_.loadFromJSON("assets/language_data.json");
snippetEngine_.loadSnippets("assets/xell_snippets.json");
```

**Key event flow (in handleEditorKeyDown):**

```
┌─ Ctrl+Space ─────────→ force show popup
│
├─ Tab (popup visible) ─→ accept selected item
│   ├─ if Snippet → expand with tab stops
│   ├─ else → insert completion text
│   └─ hide popup
│
├─ Tab (ghost visible, no popup) → accept ghost text
│
├─ Tab (snippet active) → advance to next tab stop
│
├─ Arrow Up/Down (popup visible) → navigate popup
│
├─ Escape ──────────────→ dismiss popup + ghost
│
├─ Enter (popup visible) → accept + dismiss
│
├─ Alphanumeric/_ ──────→ update prefix, re-filter
│   └─ if prefix.len >= 2 && debounce 300ms → auto-show
│
├─ Backspace ───────────→ shrink prefix, re-filter
│   └─ if prefix empty → dismiss
│
└─ Any other key ───────→ dismiss popup
```

**Rendering overlay (in renderFrame or tick):**

- After compositing editor + panels, overlay the popup cells on top
- Popup is rendered via `autocompletePopup_.render()` → blit into screenBuffer

### Phase 5: User Symbol Scanning

Already implemented in `CompletionDB::scanBuffer()`. Wire it up:

**Trigger:** On every buffer change (debounced 500ms), re-scan:

```cpp
completionDB_.scanBuffer(editor_.activeBuffer().lines());
```

**Patterns already detected by scanBuffer:**

- `fn name` → Function
- `let name` / `be name` → Variable
- `class name` / `struct name` → Class
- `module name` → Module

**Add to scanBuffer:** (expand pattern list)

- `bring name from` → imported names → Module kind
- `name = ` at line start → Variable (assignment detection)
- `for item in` → Variable (loop variable)
- `catch err` → Variable (error binding)

### Phase 6: Contextual Completion (member access)

**Trigger:** Typing `.` or `->` or `of`

After `.` / `->`:

- Look at the identifier before the dot
- If it's a known type (from assignment inference) → show members
- If unknown → show all string/list/map methods (generic)

**Lightweight type inference:**

```
x = [1, 2, 3]       → x is List   → show push, pop, sort, len...
m = {"a": 1}         → m is Map    → show keys, values, has, get...
s = "hello"          → s is String → show split, upper, lower, trim...
```

This can be a Phase 2 stretch goal — basic member suggestions.

## Rendering Layout

### Popup

```
fn greet|                     ← cursor
        ┌──────────────────────┐
        │ F greet     function │  ← selected (blue bg)
        │ B grep      textproc │
        │ B grep_regex textproc│
        └──────────────────────┘
```

- Max 12 items shown
- Kind icon (K=keyword, B=builtin, F=function, V=variable, S=snippet, C=class, M=module)
- Right-aligned category/type
- Scrollbar if more items exist

### Ghost text

```
fn gre|et_user(name) :       ← "et_user(name) :" in dim grey
```

- Shows ONLY the completion suffix (what's left after prefix)
- If top completion is a snippet, shows the first line of expanded body

### Snippet expansion

```
user types: class<Tab>
→
class |ClassName| :          ← tab stop 1 highlighted
    fn __init__(self, |params|) :    ← tab stop 2
        |body|               ← tab stop 3
    ;
;
```

- Tab stop placeholders are highlighted with a subtle background
- Tab jumps between stops
- Typing replaces the placeholder text
- Final Tab ($0 or last stop) → exit snippet mode
